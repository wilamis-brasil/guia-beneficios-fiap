# Contribuindo

Obrigado pelo interesse em contribuir com este guia. Correções de links quebrados, informações desatualizadas e melhorias de texto são bem-vindas — toda contribuição ajuda a manter o material confiável.

## 🤝 Como ajudar

### Issues

Abra uma issue ao encontrar:

- link quebrado;
- informação desatualizada em relação ao [GitHub Education](https://education.github.com/pack);
- erro de escrita;
- trecho confuso ou que pode melhorar.

Indique exatamente onde está o problema (arquivo e seção). Isso agiliza a revisão.

### Pull requests

Para corrigir diretamente, abra um pull request. De preferência:

- explique em uma frase o que mudou;
- mantenha o PR focado em um único assunto;
- revise links, texto e formatação antes de enviar.

## 📐 Guia de estilo

Para manter o guia consistente e profissional:

**Conteúdo e fontes**

- **Marca:** escreva sempre **GitHub** (G e H maiúsculos), nunca "Github".
- **Datas:** use o formato `DD/MM/AAAA` (ex.: `21/06/2026`).
- **Valores e prazos:** todo número de oferta precisa de **fonte oficial**. Sem confirmação, marque o dado com **(verificar)**.
- **Nomes de produtos:** mantenha a grafia oficial (ex.: **Codédex**, **JetBrains**, **DigitalOcean**).
- **Tom:** direto, honesto e sem sensacionalismo. Nada de "ganhe X mil garantido" — os valores citados são estimativas ilustrativas.

**Formatação**

- **Emojis:** use com moderação, como âncora visual — não como decoração.
  - Título (H1): sem emoji.
  - Seções (H2): no máximo **um** emoji inicial, opcional e consistente entre os documentos.
  - Subtítulos (H3+), listas e texto corrido: sem emoji.
  - Tabelas: sem emoji nas células. Ícone no cabeçalho apenas quando carrega significado reutilizado (ex.: a legenda `💎 / 🕒 / ⚠️` do catálogo).
- **Âncoras:** toda seção citada em um sumário recebe `<a id="kebab-sem-acento"></a>` logo acima do título; o sumário aponta para esse id. Não dependa do slug automático quando o título tem emoji ou acento.
- **Navegação:** todo documento abre com o link de retorno `[← Voltar ao README](../README.md)` e fecha com um bloco de navegação para os documentos relacionados.
- **Links internos:** use caminhos relativos e confira se as âncoras do sumário resolvem.

Como manter o conteúdo atualizado: veja [`docs/README.md`](docs/README.md#como-manter-este-guia-atualizado).
