## 🗺️ Trilhas e mapas para não estudar no escuro

<details open>
<summary><strong>📚 O que vale guardar aqui</strong></summary>

|---|---|---|---|
| [roadmap.sh](https://roadmap.sh/) | te mostra o caminho das áreas e o que costuma vir antes ou depois | quando você parar de querer estudar tudo ao mesmo tempo | não use como checklist infinita |
| [freeCodeCamp](https://www.freecodecamp.org/learn) | currículo gratuito com exercício e projeto | quando você quiser praticar com sequência | não vira certificado bom se você só cumprir etapa no automático |
| [Curso em Vídeo](https://www.cursoemvideo.com/) | apoio em português para lógica, HTML, CSS, Git e base inicial | quando você estiver travado com explicação muito seca ou inglês | use como apoio, não como muleta eterna |

</details>

## 🌐 Sites que ajudam muito na prática

<details open>
<summary><strong>🧪 Ferramentas e sites que realmente destravam</strong></summary>

| Recurso | Para que serve na prática | Quando usar | Cuidado |
|---|---|---|---|
| [MDN](https://developer.mozilla.org/en-US/docs/Learn_web_development) | aprender e consultar HTML, CSS, JavaScript e Web APIs com base séria | desde o começo, principalmente em web | não tente ler tudo corrido; use por assunto e por dúvida real |
| [CodePen](https://codepen.io/) | testar HTML, CSS e JS rápido no navegador | quando você quiser experimentar sem setup | ótimo para treino, ruim como casa final de projeto |
| [Frontend Mentor](https://www.frontendmentor.io/) | praticar front com desafios e briefs prontos | quando você já souber o básico e quiser projeto melhor | não copie solução; use para construir repertório |
| [Exercism](https://exercism.org/) | praticar linguagem com exercício e repetição | quando você já viu o básico e quer ganhar fluência | não é a melhor porta de entrada para aprender do zero |
| [GitHub Pages](https://pages.github.com/) | publicar site estático direto do repositório | quando você tiver um HTML/CSS/JS simples e quiser colocar no ar | não é backend completo |
| [JSONPlaceholder](https://jsonplaceholder.typicode.com/) | API fake para testar `fetch`, listas, cards e integração | quando você começar a consumir dados no front | por ser fake, não reproduz todas as dores de API real |

</details>

<a id="coisas-front"></a>
## 🎨 Coisas que ajudam demais no front

<details open>
<summary><strong>🖼️ Visual, estrutura e revisão sem sofrer</strong></summary>

| Recurso | Para que serve | Quando usar | Cuidado |
|---|---|---|---|
| [Pexels](https://www.pexels.com/license/) | fotos e vídeos gratuitos para deixar projeto menos vazio | quando o layout estiver sem vida | não jogue imagem aleatória só para preencher |
| [unDraw](https://undraw.co/) | ilustrações SVG leves e editáveis | quando você quiser deixar landing page, login ou portfólio mais apresentável | se usar demais, muita coisa fica com a mesma cara |
| [Lucide](https://lucide.dev/) | ícones SVG consistentes e leves | menus, botões, cards e navegação | escolha um único set por projeto |
| [Coolors](https://coolors.co/) | gerar e testar paleta de cor mais rápido | quando você travar em cor e contraste | paleta bonita não garante legibilidade boa |
| [Google Fonts](https://fonts.google.com/) | escolher fonte web sem complicação | quando o layout já estiver funcionando e você quiser melhorar a tipografia | não carregue fonte demais |
| [Bootstrap](https://getbootstrap.com/) | grid, componentes e responsividade para acelerar interface | quando você ainda não tem segurança para montar tudo do zero | se usar cru, fica com cara muito genérica |
| [HTML5 UP](https://html5up.net/) | templates responsivos para estudar estrutura de site real | quando você quiser analisar um projeto pronto ou adaptar um portfólio | entenda o código antes de sair só trocando texto |
| [Can I use](https://caniuse.com/) | ver suporte de feature nos navegadores | quando uma solução parece boa demais para ser verdade | não precisa consultar para cada linha de CSS |
| [WAVE](https://wave.webaim.org/) | encontrar problemas básicos de acessibilidade | quando a página já estiver minimamente pronta | ferramenta ajuda muito, mas não substitui revisão humana |
| [Responsively App](https://responsively.app/) | testar várias telas ao mesmo tempo | quando você começar a cuidar mais de mobile e desktop de verdade | continua sendo simulação; dispositivo real ainda importa |

</details>

<a id="ordem-boa-de-uso"></a>
## 📌 Ordem boa de uso

Se eu estivesse guiando um iniciante do jeito mais inteligente possível, faria assim:

1. usa o [roadmap.sh](https://roadmap.sh/) para entender o caminho da área;
2. usa `Curso em Vídeo` ou [freeCodeCamp](https://www.freecodecamp.org/learn) para estudar e praticar com ordem;
3. usa a [MDN](https://developer.mozilla.org/en-US/docs/Learn_web_development) para aprender a consultar documentação boa;
4. usa [CodePen](https://codepen.io/) e [Frontend Mentor](https://www.frontendmentor.io/) para praticar sem ficar só assistindo;
5. usa [GitHub Pages](https://pages.github.com/) para colocar projeto simples no ar;
6. quando começar a precisar de visual melhor, puxa `Pexels`, `unDraw`, `Lucide`, `Coolors` e `Google Fonts`;
7. quando começar integração, usa [JSONPlaceholder](https://jsonplaceholder.typicode.com/) e depois olha [public-api-lists/public-api-lists](https://github.com/public-api-lists/public-api-lists);
8. quando quiser aprofundar, consulta os repositórios e livros com calma, sem abrir dez coisas de uma vez.

Resumo sincero:

> o que mais destrói iniciante não é falta de conteúdo.
>
> é excesso de conteúdo sem ordem.

Então usa este capítulo como acervo.

Não como desculpa para sair clicando em tudo. 🚀
