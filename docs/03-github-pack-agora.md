# GitHub Student Developer Pack: o que vale ativar agora

[Voltar ao hub](../README.MD%20(2))

Este arquivo existe para evitar um erro comum no começo: querer ativar tudo de uma vez.

O GitHub Student Developer Pack faz sentido quando resolve uma dor real do seu momento. Se a ferramenta não remove atrito hoje, ela pode esperar.

> Regra simples: uma dor por vez, uma ferramenta principal por dor.

## Antes de começar

- O Pack é voltado a estudantes verificados.
- As ofertas mudam com o tempo e podem variar por parceiro, país e regra de uso.
- Se você ainda está aprendendo o básico, priorize ferramentas que ajudam a praticar, publicar e organizar sem aumentar a bagunça.

## O que faz sentido cedo

### 1. Aprender programação sem travar

**Opção principal:** [Codédex](https://education.github.com/pack)

Use quando você quer ritmo, prática guiada e uma entrada mais leve no estudo.

O Pack oferece 6 meses de Codédex Club gratuito para estudantes verificados.

**Alternativas com moderação:**

- [Scrimba](https://education.github.com/pack), se você aprende melhor vendo e editando o código no mesmo fluxo.
- [Educative](https://education.github.com/pack), se você gosta de conteúdo interativo com exercícios práticos.

**Quando vale agora:** se você ainda está formando base, precisa ganhar confiança e quer estudar sem montar um plano enorme.

**Quando não vale:** se você já está com uma trilha principal funcionando. Não abra três plataformas ao mesmo tempo.

### 2. Escrever e testar sem brigar com setup

**Opção principal:** [GitHub Codespaces](https://education.github.com/pack)

Use quando o ambiente local está travando seu avanço ou quando você precisa de um espaço pronto para codar, testar e continuar de onde parou.

O Pack dá acesso Pro ao Codespaces enquanto você usa a conta.

**Alternativas com moderação:**

- [JetBrains](https://education.github.com/pack), se você já sabe que prefere uma IDE mais completa.
- Seu editor local atual, se ele já está funcionando bem. O objetivo é diminuir atrito, não trocar de ferramenta por impulso.

**Quando vale agora:** se instalar dependência, configurar ambiente ou alternar entre máquinas está consumindo energia que deveria ir para aprender.

**Quando não vale:** se seu setup atual já está estável e você não precisa de um ambiente remoto.

### 3. Publicar um projeto simples com link próprio

**Opção principal:** [GitHub Pages](https://education.github.com/pack)

Use para portfólio estático, documentação, landing page ou demonstração simples.

No Pack, o GitHub Pages oferece um site por conta e organização, com sites de projeto ilimitados.

**Alternativas com moderação:**

- outro host estático que você já domine e que resolva o caso com menos atrito

**Quando vale agora:** assim que você tiver um projeto estático minimamente apresentável.

**Quando não vale:** se o projeto ainda muda todo dia ou se você ainda não tem nada que mereça um link público.

### 4. Organizar a semana sem inventar sistema demais

**Opção principal:** [Notion](https://education.github.com/pack)

Use quando você ainda não tem um lugar confiável para guardar:

- entregas;
- dúvidas;
- links importantes;
- backlog de estudo.

**Alternativas com moderação:**

- caderno, agenda ou Google Docs, se já estiverem funcionando de verdade

**Quando vale agora:** se a sua bagunça é mais mental e operacional do que técnica.

**Quando não vale:** se você já tem um sistema simples que realmente usa. Trocar de ferramenta por impulso não é organização.

### 5. Melhorar foco, se distração for a dor principal

**Opção principal:** [PomoDone](https://education.github.com/pack)

Use como ferramenta opcional quando o seu problema não é falta de conteúdo, e sim sentar para estudar e dispersar.

**Alternativas com moderação:**

- cronômetro simples do celular;
- Pomodoro manual em bloco de estudo.

**Quando vale agora:** se você já sabe o que precisa estudar, mas não consegue sustentar foco.

**Quando não vale:** se o problema ainda é não saber por onde começar. Nesse caso, rotina e base vêm antes.

### 6. Organizar senhas, tokens e acessos

**Opção principal:** [1Password](https://education.github.com/pack)

Use quando você começa a lidar com contas, chaves de API, tokens ou vários acessos diferentes.

O benefício dá 1 ano grátis de 1Password, incluindo as Developer Tools.

**Alternativas com moderação:**

- [Doppler](https://education.github.com/pack), se o problema principal já for gestão de segredos de projeto.
- O gerenciador de senhas do navegador, se você ainda está só no começo e não tem muita coisa sensível para controlar.

**Quando vale agora:** quando você já percebe que repetir senha ou perder token vira risco e retrabalho.

**Quando não vale:** se você ainda não tem contas importantes, chaves ou segredos para guardar.

### 7. Usar IA como apoio, não como muleta

**Opção principal:** [GitHub Copilot](https://education.github.com/pack)

Use quando você já consegue entender o que está pedindo e revisar o que a IA sugeriu.

O benefício libera Copilot Pro enquanto você é estudante.

**Alternativas com moderação:**

- Nenhuma, se a sua dificuldade principal ainda for lógica básica, sintaxe ou leitura de código.

**Quando vale agora:** quando você já escreve pequenos trechos sozinho e quer ganhar velocidade em tarefas repetitivas.

**Quando não vale:** se a IA virar substituta do raciocínio. Nesse caso, ela atrapalha mais do que ajuda.

## O que eu deixaria para depois

Deixe para a próxima fase os itens que dependem de projeto mais maduro, uso real ou uma necessidade clara:

- domínio próprio
- cloud
- banco de dados hospedado
- observabilidade
- certificação
- monetização
- ferramentas de nicho

Esses temas ficam bem detalhados no arquivo [04-o-que-deixar-para-depois.md](04-o-que-deixar-para-depois.md).
