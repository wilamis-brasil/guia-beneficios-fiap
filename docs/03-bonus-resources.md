## Leitura em voz (bookmarklet)

Ler muito na tela cansa. O repositório inclui um **bookmarklet**: salve o código como favorito, abra numa página, cole o texto e ouça (voz do sistema ou ElevenLabs, se você configurar API).

**Arquivo:** [scripts/tts-bookmarklet.js](../scripts/tts-bookmarklet.js)

> 💡 No **Microsoft Edge** a voz nativa costuma soar mais natural. Opcional: [API ElevenLabs](https://elevenlabs.io/app/developers/api-keys).

### Como instalar

<img width="1919" height="989" alt="Painel do leitor em voz" src="https://github.com/user-attachments/assets/c0d9ec4a-a99f-418e-be45-f12f730235a3" />

1. Abra [tts-bookmarklet.js](../scripts/tts-bookmarklet.js) no repositório.  
2. Copie o código **inteiro**.  
3. Crie um **novo favorito** no navegador.  
4. Cole o código no campo de URL (começar com `javascript:` é esperado).  
5. Salve com um nome claro, ex.: `Leitura em voz`.  
6. Abra a página desejada e clique no favorito.  
7. Cole o texto e use o botão de leitura no painel.  

<img width="386" height="405" alt="Recorte do painel" src="https://github.com/user-attachments/assets/12ddd63f-2196-4c73-83ff-7d776c469465" />

✅ O áudio segue o modo escolhido no painel.

## Roadmap por área

Um **roadmap** é um mapa dos assuntos de uma área e de **uma ordem possível** de estudo — não é lei. Serve para ver **onde você está** e **o que pode vir depois**.

- 🧭 [roadmap.sh](https://roadmap.sh/)

### Uso recomendado

1. Escolha a área (front, back, dados…).  
2. Olhe o mapa inteiro antes de querer cravar tudo.  
3. Pegue **um ou dois** tópicos no seu nível.  
4. Volte ao mapa depois de praticar.  

### Base comum (web)

Muita gente começa por:

- Git e GitHub  
- HTML · CSS · JavaScript
