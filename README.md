## Se você está perdido, leia isso primeiro

Você não precisa “aprender tudo” nem instalar 30 coisas. No começo, o jogo é reduzir confusão.

Pense em 4 escolhas simples:

1. Onde eu vou estudar o básico (vídeo ou interativo)?
2. Onde eu vou praticar (exercícios)?
3. Onde eu vou guardar meus projetos (GitHub)?
4. Como eu vou me organizar (anotações / tarefas)?

A maioria dos benefícios e ferramentas vira distração antes de você conseguir escrever programas simples.

### Checklist prático (sem virar “plano de estudos”)

- Crie/organize um GitHub pessoal (1 conta só).
- Peça o **GitHub Student Developer Pack** (isso destrava muita coisa).
- Ative os benefícios FIAP que dão “retorno imediato”:
  - **Alura** (se aplicável ao seu curso)
  - **Office 365** (útil para trabalho acadêmico)
  - **Nivelamento + Nano Courses** (para tapar buracos sem sofrimento)
- Escolha **1 caminho de aprendizado inicial**:
  - Vídeo guiado (ex.: Curso em Vídeo)
  - Interativo gamificado (ex.: Codédex)
  - Currículo com exercícios (ex.: freeCodeCamp)

> **Regra de ouro anti-ansiedade:** no começo, “menos opções” vale mais do que “melhor opção”.

### Glossário mínimo (para não travar em palavras)

- **IDE / Editor (VS Code):** programa onde você escreve código.
- **Git:** ferramenta que registra o histórico do seu projeto (versões).
- **GitHub:** site que hospeda seus projetos Git e facilita compartilhar.
- **Deploy / Hospedar:** colocar seu projeto no ar (acessível por link).
- **Cloud:** infraestrutura pronta (servidores/serviços) usada quando você precisa publicar algo “de verdade”.

## GitHub Student Developer Pack: os benefícios que mais valem a pena para iniciantes

### O que é (em português simples)

É um pacote de benefícios para estudantes verificados no GitHub: ferramentas profissionais e créditos educacionais que, fora disso, seriam pagos.

### Como funciona (sem mistério)

1. Você precisa ser estudante, ter conta pessoal no GitHub e comprovar vínculo.
2. Depois de aprovado, acessa o portal e ativa as ofertas que quiser.
3. Cada oferta tem regras próprias de tempo, valor e renovação.

### Por que isso é absurdamente valioso na prática

- Dá acesso a ferramentas que deixam o aprendizado menos sofrido.
- Evita gastar cedo demais com hospedagem, produtividade e ferramentas premium.

> **Uso inteligente:** não tente ativar tudo em um dia. Ative o que você vai usar agora e deixe o resto para quando tiver um projeto real.

## Como escolher sem se perder

### 1) O erro mais comum do iniciante

Confundir “ter ferramentas” com “ter habilidade”.

Ferramentas como IDE com IA, cloud, banco e deploy só começam a render quando você já consegue:

- escrever código simples sem colar tudo;
- entender erro básico;
- montar um projeto pequeno.

### 2) Uma regra que funciona: 1–1–1

Escolha:

- **1 fonte principal para aprender** (Curso em Vídeo ou Codédex ou Alura)
- **1 lugar para praticar** (freeCodeCamp, por exemplo)
- **1 jeito de organizar** (Notion, caderno ou Google Docs)

Todo o resto é opcional.

### 3) Heurística de decisão (bem objetiva)

- Se você **não entende nada e sente vergonha**: comece com vídeo (Curso em Vídeo / Alura) + exercícios leves.
- Se você **entende explicação, mas não consegue fazer sozinho**: vá para prática guiada (freeCodeCamp / Codédex).
- Se a faculdade está difícil porque **falta base**: use Nivelamento antes de se culpar.
- Se você quer **resultado visível rápido**: faça um site simples e publique no GitHub Pages.
- Se você quer **IA ajudando**: use Copilot com foco em explicação e revisão, não em “fazer por você”.

## O que priorizar agora, o que deixar para depois

### Prioridade máxima (iniciante absoluto)

- FIAP + Alura (se você tem acesso)
- Nivelamento FIAP ON quando faltar base
- GitHub Student Developer Pack
- Copilot Pro como tutor
- GitHub Pages para publicar algo simples

### Útil em seguida

- Educative / Frontend Masters / Scrimba
- Notion para organizar estudos e entregas
- Talent Lab para organização, ansiedade e carreira

### Importante mais para depois

- Azure / DigitalOcean / Heroku
- MongoDB Atlas
- Domínio (Namecheap / .TECH)

### Legal, mas não essencial agora

- Certificações cedo demais
- Repositórios avançados de arquitetura

## Conclusão com direcionamento final para iniciantes

Se você é iniciante na FIAP, o melhor uso do mundo de benefícios é este:

1. Ative o que reduz dor imediata: Alura, Nivelamento, Pack, Copilot e Pages.
2. Foque em aprender e praticar até conseguir construir um projeto pequeno sozinho.
3. Só então use cloud, hosting e créditos.

## Se eu estivesse começando hoje na FIAP sem saber nada, eu faria isso

1. Pediria o **GitHub Student Developer Pack** e ativaria **Copilot** + **GitHub Pages** primeiro.
2. Abriria o **Nivelamento** e faria o que estiver faltando para acompanhar as aulas.
3. Escolheria **um** caminho principal (Alura ou Curso em Vídeo ou Codédex) e ficaria nele até ver progresso real.
4. Publicaria um portfólio simples no GitHub Pages, mesmo com 1 projeto pequeno.
5. Se eu ficasse travado ou ansioso, marcaria o **Talent Lab** antes de pensar em desistir.
